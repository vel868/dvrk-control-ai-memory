# 18 — Next Roadmap: что делать дальше

Этот файл фиксирует дальнейший план развития DVRK Control после текущей точки проекта `v1.65-logs-integrations-settings`.

Файл предназначен для другого ИИ-агента. Его задача — показать, какие следующие шаги логичны, что делать первым, что отложить, какие ограничения соблюдать и какие зоны нельзя трогать без отдельного решения пользователя.

---

## 1. Текущая точка проекта

На момент handoff текущая рабочая точка:

```text
v1.65-logs-integrations-settings
```

Состояние:

```text
Tilda webhook работает;
заказы принимаются;
YooKassa closing receipt реализован;
Честный Знак через ZH работает;
скидки и промокоды учитываются;
orders/chz/stats/customers работают;
logs и backup работают;
WB-модуль развит;
UI settings для stats/customers работают;
/integrations создана, но ещё требует расширения.
```

---

## 2. Главный принцип дальнейшей работы

Работать нужно пошагово:

```text
одна задача;
один файл vXXX.md;
один запуск MiMo;
один report_vXXX.md;
проверка;
только потом следующая задача.
```

Не делать сразу большой пакет изменений.

Не смешивать разные направления:

```text
не делать одновременно /integrations и WB;
не делать одновременно YooKassa и UI редизайн;
не делать одновременно backup и CRM;
не делать одновременно формулы WB и Tilda webhook.
```

---

## 3. Самый логичный следующий шаг

Самый логичный следующий этап:

```text

```

Почему именно это:

```text

```

## 14. WB: чистая прибыль

Большой будущий этап:

```text
добавить внешние расходы продавца и чистую прибыль WB.
```

Сейчас WB-модуль считает:

```text
forPay;
расходы WB;
остаток после WB.
```

Но это ещё не чистая прибыль.

Для чистой прибыли нужно добавить:

```text
закупка;
доставка до WB;
фулфилмент;
упаковка;
маркировка;
реклама / ДРР;
налоговый резерв;
прочие расходы;
чистая прибыль / 1;
маржа %.
```

## 16. WB: пример massagor для чистой прибыли

Для `massagor` ранее использовались:

```text
закупка: 70 ₽;
логистика до WB: 34 ₽;
фулфилмент: 17 ₽;
себестоимость: 121 ₽;
ДРР / реклама: 105 ₽.
```

По XLSX handoff:

```text
остаток после WB: 310.16 ₽ / 1
```

До налога:

```text
310.16 − 121 − 105 = 84.16 ₽
```

После налогового резерва:

```text
примерно 49–54 ₽, в зависимости от методики.
```

---

## 17. WB: налог — осторожно

Налог УСН 6% — отдельная тема.

Нельзя без отдельной налоговой логики просто вычитать 6% от банковского перевода или от случайной суммы.

Нужно отдельно решить:

```text
какая налоговая база;
как учитывать доходы маркетплейса;
как учитывать агентскую схему;
какой управленческий резерв использовать;
как показывать налог как оценку, а не бухгалтерское заключение.
```

Лучше сначала сделать:

```text
налоговый резерв %
```

как управленческий параметр.

```text

roles;

permissions;
multi-user UI settings;

```

без отдельного запроса.

## 28. Не делать live mode YooKassa без явного подтверждения

Если YooKassa сейчас в test mode, не переключать на live mode без отдельного подтверждения пользователя.

Live mode должен иметь:

```text
явный warning;
confirm;
backup/check;
проверку credentials;

```

## 29. Не менять Tilda webhook формат

Не переводить Tilda webhook на JSON без отдельной задачи.

Стабильный формат сейчас:

```text
application/json off;
товары массивом;
после оплаты.
```

Любое изменение формата может сломать parser.

---

## 30. Не менять ZH-правило

```text
ZH = Честный Знак
```

Не заменять на другой механизм без отдельной миграции.
